Q1. What is a variable?

A variable is a named memory location used to store data.

Q2. What is a data type?

Data type specifies the type of data a variable can hold.

Q3. Difference between compiler and interpreter?

Compiler translates entire code at once; interpreter executes line by line.

Q4. What is an algorithm?

A step-by-step procedure to solve a problem.

Q5. What is debugging?

The process of finding and fixing errors in a program.

Q6. What is an array?

A collection of elements stored in contiguous memory locations.

Q7. What is a loop?

A loop executes a block of code repeatedly.

Q8. What is a function?

A reusable block of code that performs a specific task.

Q9. What is syntax?

Rules that define the structure of a programming language.

Q10. What is a keyword?

A reserved word with predefined meaning in a language.

Q11. Difference between stack and heap memory?

Stack is static and fast; heap is dynamic and slower.

Q12. What is recursion?

A function calling itself to solve smaller subproblems.

Q13. What is time complexity?

Measure of algorithm efficiency based on input size.

Q14. What is OOP?

A programming paradigm based on objects and classes.

Q15. Difference between == and = ?

= assigns value, == compares values.

Q16. What is inheritance?

One class acquiring properties of another class.

Q17. What is polymorphism?

Same function behaving differently in different contexts.

Q18. Difference between list and array?

Lists are dynamic; arrays are fixed-size.

Q19. What is exception handling?

Handling runtime errors using try-catch blocks.

Q20. What is a pointer?

A variable that stores memory address of another variable.

Q21. Difference between process and thread?

Process is heavyweight; thread is lightweight.

Q22. What is deadlock?

A state where processes wait indefinitely for resources.

TCP is reliable; UDP is faster but unreliable.

Automatic memory management that frees unused memory.

Abstract class can have methods; interface has only declarations.
